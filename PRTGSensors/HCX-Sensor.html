1. Introduction

Are you tired of manually sifting through logs and missing critical alerts in your HCX environment? Our HCX Sensor Script automates the monitoring process by retrieving alerts, categorizing their severity, and reporting the status of essential components directly to PRTG.

2. Script Overview

Purpose: The script retrieves alerts from HCX Cloud Connector/Manager and monitors Service Mesh and Interconnect statuses, outputting the information as a JSON payload for PRTG.

Key Features:

  • Fetches alerts via REST API using curl.exe.
  • Categorizes alerts into Critical, Warning, and Informational types.
  • Monitors HCX Service Mesh and Interconnect status via PowerCLI.
  • Outputs sensor data as JSON for seamless integration with PRTG.

3. Usage and Examples

Installation Instructions

  1. Clone or download the hcx_sensor.ps1 script from the GitHub repository.
  2. Ensure your environment has PowerShell, PowerCLI, and curl.exe installed.
  3. Configure your PRTG device with Windows credentials (these populate prtg_windowsuser and prtg_windowspassword environment variables).

Example Command

.\hcx_sensor.ps1 -hcx_manager <HCX_Manager_IP> -critThreshold 0.4 -warnThreshold 0.4
