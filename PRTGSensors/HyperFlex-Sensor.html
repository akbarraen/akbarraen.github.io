Cisco HyperFlex · PRTG Custom Sensor

Cisco HyperFlex Cluster Health Monitoring

Monitor Cisco HyperFlex cluster health, node state, storage health, resiliency and API availability using a PRTG custom sensor.

Solution Overview

This page provides production-focused guidance for HyperFlex-PRTG-HealthCheckSensor.ps1. The sensor is designed for PRTG EXE/Script Advanced monitoring and structured XML channel output.

Recommended sensor type: EXE/Script Advanced for multi-channel PRTG XML output.

Key Capabilities

  • Real-time health monitoring from platform API data.
  • Multi-channel PRTG output for health, alerts and response visibility.
  • Designed for production monitoring workflows.
  • Clear deployment, troubleshooting and security guidance.

Deployment Guide

  1. Download the sensor folder from GitHub.
  2. Copy HyperFlex-PRTG-HealthCheckSensor.ps1 to the PRTG probe custom sensor folder.
  3. Run the command below manually on the PRTG probe server.
  4. Confirm valid <prtg> XML output.
powershell.exe -ExecutionPolicy Bypass -File .\HyperFlex-PRTG-HealthCheckSensor.ps1

Security Considerations

  • Use least-privilege read-only monitoring credentials.
  • Do not hardcode production passwords, tokens or secrets in scripts.
  • Use PRTG credential placeholders or secure credential handling.
